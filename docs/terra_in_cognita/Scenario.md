# 시나리오

## 1. 킥요소
- 웹으로 빠른 구축 ⇒ DB베이스 ⇒ 웹앱 ⇒ 3D요소 유니티 or UE
- 모바일 ⇒ 유니티로 컨버팅
- LLM 사용 ⇒ 기획, NPC 대화

## 2. 장르
- 방치형 성장 시뮬레이션
- 영지 경영 시뮬레이션
- 텍스트 RPG, 머드게임
- TRPG (Table-Top)

## 3. 배경
### 3.1. 무한한 대지 (Terra In-Cognita)
- '지구는 둥글다' (X)
- '지구는 끝없이 넓다' (O)
- 알려진 세계보다 미지의 세계가 훨씬 넓음.
- 개척, 탐험, 미지의 자원
### 3.2. 문명 재건 (Re-Build)
- 멸망 후 재건이 아님.
- 낙후된 문명에서 발전된 문명으로.
- 고대 유적 (던전) 발굴을 통해 오파츠(Over-Technology) 획득
- 마법과 과학이 공존 (마도공학)

## 4. 플레이어
### 4.1. 역할
- 영주 (Lord), 개척자 (Pioneer), 상인 (Merchant)
- 직접 전투보다는 **관리, 지휘, 투자**에 집중
### 4.2. 목표
- 영지 확장 및 발전
- 부(Wealth) 축적
- 미지의 세계 탐험 및 지도 완성

## 5. 핵심 시스템
### 5.1. 경제 시스템
- **골드 (Gold)**: 기초 화폐. 세금, 거래 등.
- **자원 (Resources)**: 식량, 목재, 철, 마나석 등.
- **투자 (Investment)**: NPC 상단, 용병단, 마법 연구소 등에 투자하여 수익 창출.
### 5.2. NPC 상호작용
- 영지민 관리: 행복도, 세금 감면 등 정책 결정.
- **용병 고용**: 전투 대리 수행. 장비 지원.
- **상인 거래**: 특산품 교역, 물가 변동.
- NPC가 구축한 세력, 유저는 합류할 수 있음.

## 5.3. 탐험 및 전투
- 텍스트 기반 진행 (로그라이크 스타일 선택지)
- 탐험대 파견 -> 결과 보고서 수신 (방치형 요소)
- 중요 이벤트(보스전 등)에서만 직접 개입 (스킬 사용, 아이템 사용 등)

## 6. 기술 스택 (예상)
- 프론트엔드: React / Next.js (모바일 대응 반응형)
- 백엔드: Node.js / Python (FastAPI)
- DB: PostgreSQL / MongoDB (JSON 문서 저장 용이)
- AI: OpenAI API or Local LLM (NPC 대화 생성, 이벤트 생성)
