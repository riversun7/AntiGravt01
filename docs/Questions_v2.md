# Terra In-cognita: 심화 기획 질문 리스트 (v2)

웹 인터페이스 확장(네비바, 사이드바)과 내부 맵 시스템 구현을 위한 상세 질문입니다.

## 1. 인터페이스 구조 (UI/UX)
* **사이드바 구성**:
    * 사이드바에 어떤 메뉴들이 필수로 들어가야 할까요? (예: 월드맵, 내 영토 관리, 연구소, 거래소, 설정 등)
    => 좋다. 월드맵은 위치나 주요 도시들, 자원을 대략적으로 나타내고 2차맵에서 지역의 세부 타일들을 보여준다.
    현재 월드맵은 너무 작다 세계지도모양으로 땅이나 바다 강, 숲을 표현해야하고 500x500 정도의 크기가 필요하다.(더 커질예정). 세부타일은 10~20x10~20 정도의 크기가 필요하다.
    * 현재 보고 있는 '월드맵'과 '도시/내부 맵'을 전환하는 방식은 버튼식인가요, 아니면 맵을 클릭해 들어가는 방식인가요?
    => 월드맵에서 클릭을 통해 내부맵으로 전환한다. 내부맵안에 도시(소도시1x1, 중2x2, 대3x3)가 있으며 각 도시안에 건물이 배치된다. 도시의 내용이나 내부맵 타일의 정보는 새로운 탭으로 빼낸다. 세부타일탭에 생산시설이나 도시건물의 리스트가 나오고 활동을 선택할 수 있다.
* **정보 표시 (네비바/패널)**:
    * 상단 네비바에는 어떤 핵심 정보가 항상 떠 있어야 하나요? (현재: 이름, 돈. 추가: 랭킹? 알림? 턴?)
    => 아니오 적당히 보기 좋게 UI를 개선하라
    * 타일 정보는 사이드바에 표시하나요, 아니면 별도의 팝업/모달 창으로 띄우나요?
    => 사이드바 위에 답변을 참고하라

## 2. 맵의 계층 구조 (Layer System)
* **내부 맵 (Inner Map)**:
    * '타일의 내부 맵'이란 무엇을 의미하나요?
        * A: 해당 타일 안에서 다시 캐릭터가 돌아다니는 작은 로컬 맵 (예: 건물 내부, 던전)
        => 맞다. 위에 자세한 내용을 참고하라
        * B: 단순히 건물을 배치하는 슬롯 형태의 관리 화면 (심시티 스타일)
        => 어느정도 맞다 
    * 모든 타일이 내부 맵을 가지나요, 아니면 '도시'나 '기지' 같은 특정 타일만 가지나요?
    => 월드맵의 모든 타일은 내부맵을 가지고, 내부맵의 타일의 세부내용은 새로운 탭으로 빼낸다.
